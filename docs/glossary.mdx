# Glossary

<p className={styles.subtitle}>
  Technical terms and definitions used throughout this book
</p>

<div className={styles.glossaryContainer}>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Action (ROS 2)</span>
    <span className={styles.termBadge}>Communication</span>
  </div>
  <p className={styles.termDefinition}>
    A communication pattern in ROS 2 for long-running tasks that provide feedback and can be preempted. Actions combine goal request, result, and continuous feedback.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>AMCL</span>
    <span className={styles.termBadge}>Navigation</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Adaptive Monte Carlo Localization.</strong> A probabilistic localization system that uses particle filters to track a robot's pose against a known map.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Colcon</span>
    <span className={styles.termBadge}>Build Tool</span>
  </div>
  <p className={styles.termDefinition}>
    The build tool for ROS 2 workspaces. Replaces catkin from ROS 1.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>cuVSLAM</span>
    <span className={styles.termBadge}>NVIDIA</span>
  </div>
  <p className={styles.termDefinition}>
    NVIDIA's CUDA-accelerated Visual Simultaneous Localization and Mapping implementation in Isaac ROS.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>DDS</span>
    <span className={styles.termBadge}>Middleware</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Data Distribution Service.</strong> The middleware standard underlying ROS 2 communication, providing publish-subscribe messaging.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Digital Twin</span>
    <span className={styles.termBadge}>Simulation</span>
  </div>
  <p className={styles.termDefinition}>
    A virtual representation of a physical robot that mirrors its real-world counterpart in simulation.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Gazebo</span>
    <span className={styles.termBadge}>Simulation</span>
  </div>
  <p className={styles.termDefinition}>
    An open-source 3D robotics simulator with physics engine, sensor simulation, and ROS integration. Gazebo Harmonic is the version paired with ROS 2 Jazzy.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>IMU</span>
    <span className={styles.termBadge}>Sensor</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Inertial Measurement Unit.</strong> A sensor that measures acceleration and angular velocity, essential for robot balance.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Isaac ROS</span>
    <span className={styles.termBadge}>NVIDIA</span>
  </div>
  <p className={styles.termDefinition}>
    NVIDIA's hardware-accelerated ROS 2 packages for perception, including cuVSLAM, nvblox, and Freespace Segmentation.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Isaac Sim</span>
    <span className={styles.termBadge}>NVIDIA</span>
  </div>
  <p className={styles.termDefinition}>
    NVIDIA's photorealistic simulation platform built on Omniverse, designed for AI robot training and testing.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>LiDAR</span>
    <span className={styles.termBadge}>Sensor</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Light Detection and Ranging.</strong> A sensor that measures distances using laser pulses to create 3D point clouds.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>LLM</span>
    <span className={styles.termBadge}>AI</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Large Language Model.</strong> In VLA pipelines, LLMs translate natural language commands into robot action sequences.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Nav2</span>
    <span className={styles.termBadge}>Navigation</span>
  </div>
  <p className={styles.termDefinition}>
    The Navigation 2 stack for ROS 2, providing path planning, localization, and obstacle avoidance.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Node</span>
    <span className={styles.termBadge}>ROS 2 Core</span>
  </div>
  <p className={styles.termDefinition}>
    The basic executable unit in ROS 2 that performs computation and communicates via topics, services, and actions.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>nvblox</span>
    <span className={styles.termBadge}>NVIDIA</span>
  </div>
  <p className={styles.termDefinition}>
    NVIDIA's real-time 3D reconstruction library for building occupancy maps from depth sensors.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Publisher</span>
    <span className={styles.termBadge}>Communication</span>
  </div>
  <p className={styles.termDefinition}>
    A ROS 2 entity that sends messages on a topic.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Python rclpy</span>
    <span className={styles.termBadge}>ROS 2 Client</span>
  </div>
  <p className={styles.termDefinition}>
    The Python client library for ROS 2, providing APIs for nodes, publishers, subscribers, services, and actions.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>ROS 2</span>
    <span className={styles.termBadge}>Framework</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Robot Operating System 2.</strong> A set of software libraries and tools for building robot applications. Version Jazzy is the LTS release supported until 2029.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>SDF</span>
    <span className={styles.termBadge}>Format</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Simulation Description Format.</strong> An XML format for describing robots and environments in Gazebo, more expressive than URDF for simulation properties.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Service</span>
    <span className={styles.termBadge}>Communication</span>
  </div>
  <p className={styles.termDefinition}>
    A synchronous request-response communication pattern in ROS 2.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Subscriber</span>
    <span className={styles.termBadge}>Communication</span>
  </div>
  <p className={styles.termDefinition}>
    A ROS 2 entity that receives messages from a topic.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>TF2</span>
    <span className={styles.termBadge}>Transform</span>
  </div>
  <p className={styles.termDefinition}>
    The ROS 2 transform library that maintains relationships between coordinate frames over time.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Topic</span>
    <span className={styles.termBadge}>Communication</span>
  </div>
  <p className={styles.termDefinition}>
    A named bus over which nodes exchange messages in ROS 2's publish-subscribe pattern.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>URDF</span>
    <span className={styles.termBadge}>Format</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Unified Robot Description Format.</strong> An XML format for describing robot kinematics, dynamics, and visual elements.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>VLA</span>
    <span className={styles.termBadge}>AI</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Vision-Language-Action.</strong> An AI architecture that connects visual perception, language understanding, and robot actions.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>VSLAM</span>
    <span className={styles.termBadge}>Perception</span>
  </div>
  <p className={styles.termDefinition}>
    <strong>Visual Simultaneous Localization and Mapping.</strong> Using camera images to simultaneously build a map and localize within it.
  </p>
</div>

<div className={styles.termCard}>
  <div className={styles.termHeader}>
    <span className={styles.termName}>Whisper</span>
    <span className={styles.termBadge}>AI</span>
  </div>
  <p className={styles.termDefinition}>
    OpenAI's automatic speech recognition model used for voice-to-text in VLA pipelines.
  </p>
</div>

</div>
